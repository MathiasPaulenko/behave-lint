# I18N Rules

Rules that detect internationalization issues in Gherkin feature files,
such as hardcoded locale-specific text or non-ASCII characters.

**Default severity:** WARNING

| Rule | Name | Severity | Auto-fix |
|------|------|----------|----------|
| BI18N001 | [hardcoded-date-format](#bi18n001-hardcoded-date-format) | WARNING | No |
| BI18N002 | [hardcoded-currency](#bi18n002-hardcoded-currency) | WARNING | No |
| BI18N003 | [non-ascii-step-text](#bi18n003-non-ascii-step-text) | INFO | No |

## BI18N001: hardcoded-date-format

Detects steps that contain hardcoded date or time formats. Date formats
are locale-specific and may cause failures in different regions.

### Example

```gherkin
# Bad
Given the date is "12/31/2024"

# Good
Given the date is "<booking_date>"
```

## BI18N002: hardcoded-currency

Detects steps that contain hardcoded currency symbols like `$`, `€`, or
`£`. Currency symbols are locale-specific.

### Example

```gherkin
# Bad
Given the total is "$99.99"

# Good
Given the total is "<amount>"
```

## BI18N003: non-ascii-step-text

Detects steps that contain non-ASCII characters. While UTF-8 is widely
supported, non-ASCII characters can cause encoding issues in some
environments.

### Example

```gherkin
# Bad
Given the customer selects "café au lait"

# Good
Given the customer selects "cafe au lait"
```
